The Original Website (Before)

The original mackmagnets.com was a standard Shopify storefront using a basic theme. While Shopify is excellent for e-commerce, the default theme left little room for brand storytelling — the site felt like a generic product catalog rather than a warm, personality-driven Sacramento small business.

What the old site had

❌ Limitations
  • Default Shopify theme — minimal customization
  • Only 1 product listed — limited catalog
  • No dedicated About page telling the brand story
  • No Events page (event magnets are a major revenue stream)
  • No Wholesale inquiry page
  • No FAQ page — common questions unanswered
  • No SEO structured data (JSON-LD schemas)
  • No shipping, refund, privacy, or terms policies
  • No sitemap.xml or robots.txt

✓ What it did have
  • Working product page with basic listing
  • Shopify checkout & payment processing
  • Basic contact form
  • Custom domain: mackmagnets.com
  • SSL certificate
📊 Key Takeaway
The old site worked as a minimal shop, but it was missing the brand experience — the events page, the wholesale funnel, the founder's story, product categories, and the SEO infrastructure needed for organic growth in the Sacramento area and beyond.

Why these colors?

Warm Coral (#E07A5F) is the primary brand color. It was chosen because Mack Magnets is a family-run Sacramento business with a warm, creative, handcrafted feel. Coral communicates energy, approachability, and fun — perfect for a magnet company that's all about spreading joy. It's visible enough to grab attention without being overwhelming, and pairs beautifully with the other earth tones.

Sage Green (#81B29A) provides natural contrast. It's used for secondary CTAs, success states, and elements that need to stand apart from primary coral buttons. Sage + coral is a classic pairing in lifestyle branding — it feels earthy, handmade, and Sacramento-friendly.

Golden Sand (#F2CC8F) is the accent used for badges, highlights, and hover states. It adds warmth and a subtle premium feel that reflects California golden-hour lighting — which matches the vibe of Mack Magnets' handmade products.

Warm Cream (#FDF6EC) replaces harsh white as the background color. Pure white feels clinical — cream feels warm, inviting, and "homey," which matches the brand's personality perfectly.

Typography

RoleFontWhy
Headings Playfair Display Elegant serif with personality. Feels editorial and refined but not stuffy — perfect for a creative brand.
Body text Inter Highly legible at all sizes, excellent on screens, and feels modern. The gold standard for UI sans-serifs.
Accents Cormorant Garamond Used for hero subtitles and pull quotes. Adds handwritten elegance that complements the serif + sans combo.

Overall design philosophy

The design follows a "warm minimalism" approach. The site should feel like walking into a cozy, well-lit craft studio in Sacramento — not a Silicon Valley startup page and not a cluttered marketplace. Every design choice supports three goals:

  1. Trust — clean layout, consistent spacing, professional typography
  2. Warmth — cream backgrounds, soft colors, rounded corners, gentle animations
  3. Conversion — clear CTAs, product-first layout, minimal friction to contact/inquiry

Technology Stack & Why We Chose It

HTML5 + CSS3 + Vanilla JS Zero build step, zero framework. Just files.
GitHub Pages Free hosting, CDN, SSL, deploys on git push
CSS Custom Properties Full design system with 40+ tokens
Google Fonts Playfair Display, Inter, Cormorant Garamond
Schema.org (JSON-LD) Organization, LocalBusiness, FAQPage
Intersection Observer API Scroll-triggered animations & lazy loading

Why vanilla HTML/CSS/JS over React, Next.js, etc.?

✓ Pros of our approach

  • Zero build step — edit HTML, push to GitHub, done
  • No Node.js, npm, webpack, or framework updates to maintain
  • Loads in <1 second — no JavaScript framework overhead
  • Anyone can edit — no React/TypeScript knowledge needed
  • GitHub Pages = $0/month hosting forever
  • No dependency vulnerabilities to patch
  • Perfect Lighthouse scores achievable

✗ Trade-offs

  • Repeating navbar/footer HTML across page files
  • No component reuse (no <Header /> component)
  • Adding a new page means copying boilerplate
  • CSS file is large (~2,000+ lines) — no CSS modules
  • No server-side rendering or dynamic routes
💡 Why this was the right call
For a small business with ~14 pages, the simplicity wins outweigh the trade-offs. There are no API routes to maintain, no Node runtime to keep updated, no build pipeline to break. Any developer can open any HTML file, understand it immediately, and make changes. The site will still work exactly the same in 5 years with zero maintenance.

SEO & Accessibility

Search engine optimization

Every page is fully optimized for search engines with both technical and on-page SEO:

Before
  • Generic Shopify meta tags
  • No structured data (JSON-LD)
  • No custom sitemap
  • No robots.txt
  • No Open Graph or Twitter Card tags
  • No local SEO for Sacramento area
After
  • Custom meta title + description on every page
  • Organization, LocalBusiness & FAQPage schemas
  • Custom sitemap.xml with page priorities & frequencies
  • Proper robots.txt with sitemap reference
  • Full OG + Twitter Card meta on every page
  • Sacramento-local keywords & business info

Structured data example

JSON-LD { "@context": "https://schema.org", "@type": "LocalBusiness", "name": "Mack Magnets", "description": "Handcrafted fridge magnets, Sacramento CA", "url": "https://mackmagnets.com", "address": { "@type": "PostalAddress", "addressLocality": "Sacramento", "addressRegion": "CA" } }

Accessibility (WCAG 2.1 AA)

The site was built with accessibility as a first-class concern, not an afterthought:

  • Keyboard navigation — every interactive element is reachable via Tab/Enter/Space
  • Skip links — "Skip to main content" on every page for screen reader users
  • ARIA labels — all buttons, nav elements, and landmark regions are labeled
  • Color contrast — all text meets WCAG AA contrast ratios (4.5:1 minimum)
  • Semantic HTML — proper heading hierarchy, landmarks, lists, and form labels
  • Focus indicators — visible focus rings on all interactive elements
  • Alt text — descriptive alt attributes on all images
✅ axe-core audits
All 8 pages tested with the axe-core accessibility engine — 0 violations across the entire site. This means screen readers, keyboard users, and assistive technology all work perfectly.
